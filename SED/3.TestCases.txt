**AIM**
To design test cases for a banking application.

---

**THEORY**
A test case is a set of conditions used to verify the functionality of an application.
It includes input, execution steps, and expected output.
Test cases help ensure software correctness and detect errors.

Components of Test Case:

* Test Case ID
* Description
* Pre-condition
* Test Steps
* Expected Result

---

**TEST CASES**

**1. LOGIN MODULE**

```text id="tc1"
TC_LG_01 | Verify login with valid username and password | User account exists | 
1. Open login page 
2. Enter valid username 
3. Enter valid password 
4. Click login | 
User should successfully log in
```

```text id="tc2"
TC_LG_02 | Verify login with invalid password | User account exists | 
Enter valid username and invalid password | 
Error message "Incorrect Password"
```

```text id="tc3"
TC_LG_03 | Verify login with blank fields | - | 
Click login without entering data | 
Validation message should appear
```
